# PhASAR-advisor

Инструмент рекомендаций по LLVM IR с помощью фреймворка PhASAR.

## Доступные анализы

1. **Basic Info** (`info`) - Базовая информация об IR
   - Статистика модулей и функций
   - Количество блоков и инструкций в функциях

2. **Call Graph** (`callgraph`) - Граф вызовов
   - Построение графа вызовов функций
   - Статистика прямых и косвенных вызовов
   - Анализ цепочек вызовов и рекурсии
   - Экспорт в формат DOT для визуализации
   - [Подробная документация](CALLGRAPH_README.md)

3. **Constant Propagation** (`const`) - Распространение констант
   - Обнаружение константных значений в коде
   - Поиск арифметических операций с константами
   - Анализ возможностей оптимизации

4. **Uninitialized Variables** (`uninit`) - Неинициализированные переменные
   - Обнаружение чтения неинициализированных переменных
   - Поиск неиспользуемых переменных (мёртвый код)
   - Обнаружение переменных, которые записываются, но не читаются

5. **Type Analysis** (`types`) - Анализ типов
   - Подсчет приведений типов
   - Анализ иерархии типов и наследования
   - Определение подтипов для каждого класса

6. **Alias Analysis** (`alias`) - Анализ алиасов
   - Обнаружение указателей, ссылающихся на одну и ту же память
   - Анализ отношений между указателями
   - Определение потенциальных алиасов
   - Анализ доступа к памяти

7. **Code Complexity** (`complexity`) - Анализ сложности кода
   - Расчет цикломатической сложности функций
   - Определение глубины вложенности
   - Подсчет инструкций, базовых блоков и точек ветвления
   - Рекомендации по упрощению кода при превышении пороговых значений

## Сборка

```bash
mkdir build && cd build
cmake ..
ninja  # или make
```

## Использование

### Синтаксис
```bash
./PhASAR-advisor <llvm_ir_file> [analyses...]
```

### Доступные анализы
- `info` - Basic IR information
- `callgraph` - Call graph analysis (граф вызовов)
<!-- - `taint` - Taint analysis -->
- `const` - Constant propagation
- `uninit` - Uninitialized variables
- `types` - Type analysis
- `alias` - Alias analysis (анализ алиасов)
- `complexity` - Code complexity analysis (анализ сложности)
- `all` - Все доступные анализы

## Требования

- LLVM 16
- PhASAR framework
- CMake 3.12+
- C++20 совместимый компилятор

## Расширение функциональности

Для добавления новых анализов:

1. Добавить новый тип анализа в enum `AnalysisType`
2. Добавить соответствие в `ANALYSIS_MAP`
3. Создать функцию анализа вида `runNewAnalysis(LLVMProjectIRDB& IRDB)`
4. Добавьте case в switch statement в main функции
